# 15Loop 개발기 소스팩 추출 브리프 (codex exec 실행용)

> **완료 안내 (2026-07-19):** 이 추출 브리프의 결과물은 `docs/BUILD_STORY_SOURCE.md`에 커밋·파일 근거 중심 소스팩으로 생성됐고, `docs/handoff/02_BUILD_STORY_MASTER.md`에서 마케팅용 상세 개발기로 정리됐다. 실제 마케팅 실행은 `docs/handoff/03_MARKETING_CONTENT_HANDOFF.md`와 Project JYP의 `15Loop_개발기_마케팅_HANDOFF_2026-07-19.md`를 사용한다.

너는 이 레포(15Loop — 초5·6/중1 영어 단어 연결 진단·학습 서비스)의 **개발 전 과정을 스레드 연재의 스토리 재료로 채굴**한다. 목적: "만들어서 파는 연재"(아빠가 AI로 아이 영어 앱을 만들어 실제 매출까지 가는 과정 공개)의 1차 소스팩.

## 원칙
- **사실만.** git 히스토리·문서·코드에 근거 없는 이야기를 지어내지 않는다. 각 항목에 근거(커밋 해시·파일 경로)를 표기한다. 각색·문체는 다음 단계(Claude+사람) 몫 — 너는 재료만 정확히.
- 에피소드로 쓸 수 있는 "장면" 단위로 추출한다 (기능 나열이 아니라 결정·갈등·전환점 중심).

## 소스
`git log --stat` 전체(브랜치·머지·PR 제목 포함), `docs/*`, `README.md`, `OPEN_BETA_KPI.md`, `db/schema.ts`, 주요 디렉토리 구조 변화, 커밋 메시지의 리버트·수정 흔적.

## 추출 항목 (각각 섹션으로)
1. **타임라인** — 첫 커밋~현재: 날짜별 주요 마일스톤 (몇 일 걸렸나, 주차별 뭐가 생겼나). 총 커밋 수·개발 일수 같은 검증 가능한 숫자 포함.
2. **결정의 순간들** — 제품 철학이 드러나는 결정과 그 근거: 예) 진단을 가입 없이 시작하게 한 것, "아이에게 계정을 만들게 하지 않는다", 가격 12,900원/7일 무료 설계, 스택 선택(Next.js·Cloudflare D1·Supabase Auth 혼합), 30단어 검수 풀 시작. 각 결정이 어느 커밋/문서에서 확인되는지.
3. **실패·수정 로그** — 뒤집은 것, 버린 것, 다시 만든 것 (리버트·대규모 리팩터·"fix"류 반복 흔적). 연재에서 신뢰 콘텐츠(@semoji 포맷)로 쓸 재료.
4. **숫자 팩트 시트** — 총 커밋, 개발 기간, 마이그레이션 수, 테스트 수, 기능 수, 단어 풀 크기 등 **검증 가능한 것만** (스크린샷/명령으로 재현 가능해야 함).
5. **AI 협업 흔적** — Codex/Claude/에이전트가 만든 부분의 증거: 브랜치명(예: agent/*), docs/codex/, PR 제목 등. "AI와 같이 만들었다" 스토리의 팩트 근거.
6. **아빠 서사 연결점** — 랜딩의 "한 아빠의 발견" 서사와 맞닿는 제품 요소 (4개 연결 점수 진단, 뜻·소리·문장 연결 설계 등)가 코드 어디에 어떻게 구현돼 있는지.
7. **에피소드 후보 리스트** — 위 재료를 조합해 "연재 에피소드로 쓸 만한 장면" 10~15개를 한 줄 제목 + 근거로 제안 (예: "가입 버튼을 없앤 날 — 진단을 무가입으로 바꾼 커밋").

## 출력
- 파일: `docs/BUILD_STORY_SOURCE.md` (이 레포에 저장)
- 완료 후 콘솔에: 파일 경로 + 에피소드 후보 수 + 흥미로운 발견 톱3 요약.
