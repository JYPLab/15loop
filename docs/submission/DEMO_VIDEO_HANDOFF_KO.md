# 15Loop 해커톤 데모 영상 촬영 핸드오프

이 문서는 자체완결형 실행 지시서다. 촬영 담당자는 다른 대화 기록을 읽지 않아도 이 문서만으로 15Loop 데모 영상을 녹화·편집·업로드하고 Devpost 제출 담당자에게 전달할 수 있어야 한다.

## 1. 목표와 현재 단계

목표는 OpenAI Build Week Education 트랙 제출용 **3분 미만 공개 데모 영상**을 완성하는 것이다.

제품 개발과 제출 문서 준비는 완료됐다. 지금 단계는 새로운 기능 개발이 아니라 실제 운영 제품을 짧고 정확하게 보여주는 촬영·편집·업로드 단계다.

- 제품: 15Loop
- 운영 URL: https://15loop.com
- 공개 저장소: https://github.com/JYPLab/15loop
- 제출 트랙: Education
- 영상 언어: 영어 설명 또는 영어 자막 필수
- 권장 길이: 2분 45초~2분 55초
- 절대 상한: 3분 미만
- 제출 마감: 2026년 7월 22일 오전 9시 KST

현재 GitHub `main`에는 검색·분석 기반과 제출 패키지가 PR #4로 머지되어 있다.

- PR: https://github.com/JYPLab/15loop/pull/4
- 머지 커밋: `0cff5c7d59a126c18caa2d8b4379a618067e12d8`
- 검증: 린트 통과, 빌드 통과, 자동 테스트 42개 통과
- 운영 확인: 진단 페이지, `robots.txt`, `sitemap.xml` 모두 HTTP 200

## 2. 역할 구분

### 사람이 반드시 해야 하는 일

1. 실제 운영 화면을 직접 조작하며 화면 녹화를 시작하고 종료한다.
2. 음성을 직접 녹음하거나 사용할 음성을 최종 승인한다.
3. 영상 전체를 직접 확인해 개인정보와 오류가 없는지 승인한다.
4. 본인 YouTube 계정에 영상을 공개 또는 일부 공개로 업로드한다.
5. Devpost의 소유권·동의 항목을 확인하고 최종 제출한다.

얼굴 출연은 필요 없다. 실제 제품 화면과 목소리 또는 영어 자막만으로 충분하다.

### AI/Codex가 지원할 수 있는 일

- 대본과 영어 문장 수정
- 자막 원고 정리
- 촬영 순서와 컷 길이 조정
- 제출 설명문 작성
- 공개 전 링크·문구·기술 설명 검토

## 3. 변경하면 안 되는 제품 메시지

- 15Loop는 단순히 단어 뜻을 맞혔는지 확인하지 않는다.
- 단어의 글자·뜻, 소리, 문맥, 떠올리기 연결을 나누어 평가한다.
- 무료 진단이 부모 가입보다 먼저다.
- 계속 학습하려면 부모가 로그인하고 아이 프로필을 연결한다.
- 학습 목표는 하루 30단어가 아니라 **15분 동안 실제로 연결한 단어 수**다.
- GPT-5.6은 제한된 학습 과제의 의미와 사용을 평가한다.
- 학습 성적 향상이나 기억 효과가 이미 입증됐다고 주장하지 않는다.
- 결제 기능은 이번 심사 경로에 포함하지 않는다.

## 4. 촬영 전 준비

### 기기와 화면

- 권장 브라우저: Chrome 시크릿 창 또는 개인정보가 없는 새 프로필
- 권장 제품 화면: 모바일 뷰포트 430 × 932
- 권장 영상 캔버스: 1920 × 1080
- macOS 기본 녹화: `Shift + Command + 5` → 선택 영역 기록
- 알림, 메신저, 자동완성, 북마크 바를 모두 숨긴다.
- 브라우저 확대율은 글자가 영상에서 읽히는 수준으로 맞춘다.

### 데이터와 계정

- 무료 진단은 새 시크릿 창에서 가입 없이 촬영한다.
- 부모 화면이 필요하면 실명이 아닌 일반 테스트 프로필만 사용한다.
- 실제 아이 이름, 부모 이메일, 학교명, 프로필 ID를 노출하지 않는다.
- 비밀번호, API 키, 쿠키, 개발자 도구의 네트워크 헤더를 촬영하지 않는다.

### 촬영 직전 점검

- [ ] https://15loop.com 이 정상적으로 열린다.
- [ ] 무료 진단 버튼이 가입 없이 시작된다.
- [ ] 듣기 버튼에서 영어 음성이 나온다.
- [ ] 결과 화면까지 흐름이 정상 작동한다.
- [ ] 실제 GPT-5.6 평가가 반환되고 출처 라벨이 보인다.
- [ ] 화면에 개인 계정 정보와 알림이 없다.
- [ ] 마이크 테스트에서 목소리가 또렷하게 녹음된다.

## 5. 권장 촬영 순서

전체 진단 20~25개를 영상에서 실시간으로 전부 진행하지 않는다. 실제 제품을 사용하되 대표 장면 사이를 편집한다. 컷 편집이 전체 진단이 짧은 것처럼 오해를 만들지 않도록 진행 표시를 남긴다.

| 시간 | 화면 | 전달할 핵심 |
|---|---|---|
| 0:00–0:18 | 랜딩 화면 | 아들이 중학교에 들어간 뒤 발견한 실제 단어 학습 문제 |
| 0:18–0:38 | 무료 진단 시작 | 부모 가입이나 결제보다 아이의 무료 경험이 먼저 |
| 0:38–1:04 | 네 가지 대표 문제 | 글자·뜻, 듣기, 문맥, 떠올리기를 분리 평가 |
| 1:04–1:25 | 진단 결과 | 아이를 평가절하하지 않고 약한 연결을 먼저 설명 |
| 1:25–1:52 | 15분 학습 | 약한 연결 우선, 틀린 단어의 지연 반복, 학습 단어 수 |
| 1:52–2:18 | GPT-5.6 평가 | 문자열 일치가 아닌 제한된 의미·사용 평가와 이중언어 피드백 |
| 2:18–2:39 | GitHub와 테스트 | Codex 협업, 구현 기록, 자동 테스트 |
| 2:39–2:53 | 결과와 로고 | 대상 학습자와 장기적인 평가 엔진 비전 |

## 6. GPT-5.6 촬영 장면

이 장면은 영상의 기술적 핵심이므로 실제 운영 요청이 돌아오는 과정을 한 번은 끊지 않고 보여준다.

권장 테스트:

- 목표 단어: `borrow`
- 입력 문장: `Could I borrow your pencil until class ends?`

확인할 내용:

- 로딩 상태가 보인다.
- GPT-5.6 평가 출처 라벨이 보인다.
- 한국어와 영어 피드백이 읽힌다.
- 개인 이름이나 이메일이 화면에 없다.
- API 오류가 발생하면 그 장면을 사용하지 말고 원인을 확인한 뒤 다시 촬영한다.

로컬 평가로 대체됐는데 GPT-5.6 결과라고 설명하면 안 된다. 화면이 `local fallback`을 표시하면 해당 컷을 GPT-5.6 장면으로 사용하지 않는다.

## 7. 내레이션 원칙

영어 내레이션 원문은 `docs/submission/DEMO_SCRIPT.md`를 사용한다.

직접 영어로 말하기 부담스러우면 다음 순서로 진행한다.

1. 화면만 깨끗하게 녹화한다.
2. 한국어 내레이션을 별도로 녹음한다.
3. 정확한 영어 자막을 넣는다.

영어 자막은 제품의 한국어 버튼을 처음 사용할 때 뜻을 함께 보여준다.

- `우리 아이 무료 진단하기` → `Start my child’s free diagnostic`
- `부모에게 결과 연결` → `Connect this result to a parent account`
- `뜻 보고 단어 떠올리기` → `Recall the word from its meaning`

배경음악은 필요 없다. 사용한다면 저작권이 확실한 음원만 쓰고 설명 음성을 방해하지 않도록 매우 작게 유지한다.

## 8. 편집 기준

- 최종 영상은 3분 미만이어야 한다.
- 모바일 화면의 글자가 데스크톱 플레이어에서도 읽혀야 한다.
- 한국어 UI의 핵심 의미는 영어 자막으로 설명한다.
- GPT-5.6 피드백 화면은 최소 3초 동안 읽을 수 있게 둔다.
- 전체 진단이 몇 문항 안 되는 것처럼 보이게 편집하지 않는다.
- 제품 오류, 로딩 실패, 개인정보가 보이는 컷은 삭제한다.
- 비공개 관리자 화면이나 환경변수 화면을 넣지 않는다.
- 영상 마지막에는 아래 문구를 표시한다.

```text
15Loop
15 minutes. Words that stay.
https://15loop.com
```

## 11. Devpost 제출 담당자에게 전달할 것

촬영 담당자는 다음 세 가지를 전달한다.

1. 공개 또는 일부 공개 YouTube URL
2. 최종 MP4 원본 파일 위치
3. 촬영 중 발견한 제품 오류나 영상에서 생략한 기능 목록

Devpost 제출 담당자는 YouTube URL을 `docs/submission/DEVPOST_COPY.md`의 `[ADD PUBLIC YOUTUBE URL]` 자리에 넣는다.

영상 외에 아직 사람의 판단이 필요한 제출 항목:

- 정확한 Codex `/feedback` Session ID 입력

- Devpost 소유권·동의 확인
- 최종 Submit 실행
