# 15Loop 결정 로그와 미결정 항목

*목적: 과거 아이디어가 다시 섞여 제품·마케팅이 흔들리지 않게 하는 변경 관리 정본*

## 1. 확정된 결정

| 영역 | 확정 결정 | 이유 |
|---|---|---|
| 제품명 | 15Loop | 15분 학습과 반복 엔진을 동시에 표현, 기존 LoopVoca 충돌 회피 |
| 대상 | 초5·6, 중1 | 첫 오픈베타에서 문제와 난이도를 좁힘 |
| 제품 본질 | evaluation | 콘텐츠 양보다 약한 연결과 다음 복습 결정이 차별점 |
| 연결 | 글자·뜻, 듣기, 문맥, 떠올리기 | 단일 ‘단어를 안다’ 점수보다 실제 약점 설명 가능 |
| 일일 목표 | 15분 | 고정 30단어보다 난이도와 개인 속도에 공정함 |
| 첫 경험 | 무가입 무료 진단 | 아이가 가치 확인 후 부모 연결 |
| 진단 길이 | 20개 + 약한 연결 5개 | 10개 한 번보다 연결 패턴 확인에 충분한 베타 구조 |
| 계정 | 부모 소유 | 아이 이메일·비밀번호·코드 제거, 최대 3명 분리 관리 |
| 무료 프로그램 | 7일, 결제정보 없음 | 한 번의 진단보다 반복 사용을 먼저 검증 |
| 가격 가설 | 자녀 1명 30일 12,900원 | 모집 전에 공개하고 실제 행동 후 의향 측정 |
| 가격 질문 | 완료 학습일 3일 후 | 응원성 설문이 아닌 사용 행동과 결합 |
| recall | 한 단어 cloze 필수, 자유문장 선택 | 실제 아이가 문장 전체 쓰기를 부담으로 느낌 |
| AI | 제한 평가 중심 | 생성량보다 의미 판단에 강점, 안전·품질 통제 |
| 오답 반복 | 다른 단어 뒤 지연 재출제 | 직전 답 기억을 습득으로 착각하지 않음 |
| 콘텐츠 | 30개 공개, 3,000 지도·400 후보는 검수 대기 | 양보다 교육·권리·예문 품질 우선 |
| 소셜 | 비공개 5단어 도전 | 재미는 유지하고 공개 서열·개인정보 회피 |
| 분석 | GA4는 퍼널, DB는 학습·유지 정본 | 아동 학습 데이터를 마케팅 플랫폼에서 분리 |
| 결제 | 오픈베타·해커톤에서 제외 | 법률·PG·환불·보호자 확인 검토 필요 |

## 2. 폐기되거나 대체된 아이디어

| 이전 아이디어 | 현재 상태 | 대체 결정 |

| 가입 후 무료 경험 | 폐기 | 무가입 진단 후 부모 연결 |

| `인출` 부모 카피 | 폐기 | `떠올리기` |

## 3. 지표 간 충돌 해소

### 제품 KPI

- 30 적합 가정
- 10가정씩 세 코호트
- WPA 5가정

### 콘텐츠 도전 KPI

- ‘베타 1,000명’은 Threads의 공개 미션·도달 장치로 운영 가능
- 제품 성공은 1,000 가입이 아니라 적합 가정의 반복 사용과 WPA로 판정

### Project JYP KPI

- 개발기를 보고 프로젝트 브리프·첫 빌드 행동을 한 사람
- WAB 6명

같은 숫자표에 세 KPI를 합산하지 않는다.
