# 15Loop 오픈베타 운영 핸드오프

*목적: 첫 적합 가정을 모집하고 7일 사용·가격 의향을 검증하는 운영 정본*

## 1. 운영 목표

무료 사용자를 최대한 많이 모으는 것이 아니다. 아이가 실제로 15분 학습을 반복하고, 그 모습을 본 부모가 30일 12,900원을 검토하는지 확인한다.

### 4주 목표

| 단계 | 목표 |
|---|---:|
| 가격을 본 적합 부모 | 30가정 |
| 무료 진단 완료 | 20가정 |
| 진단을 아이 프로필에 연결 | 18가정 |
| 첫 학습 | 15가정 |
| 서로 다른 3일의 15분 학습 완료 | 10가정 |
| 7일 프로그램 완료 | 6가정 |
| WPA | 5가정 |
| 부모 인터뷰 | 5가정 |

## 2. 모집 대상

포함:

- 초등 5·6학년 또는 중학교 1학년 자녀
- 뜻은 외우지만 읽기·듣기·문장·떠올리기 중 어려움이 있다고 느끼는 가정
- 부모가 7일 동안 최소 3일 사용 여부를 확인할 수 있음
- 가격 12,900원을 미리 보고도 테스트할 의향이 있음
- 사용 후 짧은 인터뷰나 피드백 의향이 있음

제외:

- 현재 학년 범위 밖
- 부모가 자녀 대신 답을 입력
- 무료 자료 수집만 목적
- 단기간 성적 보장 요구
- 가격을 숨겨야만 참여할 의향이 있는 경우

## 3. 코호트 운영

첫 30가정은 10가정씩 세 코호트로 나눈다.

| 코호트 | 목적 | 시작 조건 |
|---|---|---|
| A, 1~10가정 | 흐름·오류·진단 길이 발견 | 운영자 직접 연락 가능 |
| B, 11~20가정 | A에서 수정한 회복·부모 안내 확인 | A의 치명적 오류 해결 |
| C, 21~30가정 | 가격 의향과 반복 사용 재현 | 핵심 퍼널 안정 |

한 번에 30가정을 모두 열지 않는다. A의 오류를 B와 C에 그대로 반복시키지 않는다.

## 4. 사용자 여정과 운영 확인

### D0

1. UTM 링크 방문
2. 무가입 진단 시작·완료
3. 결과 이해 여부 확인
4. 부모 계정 연결
5. 보호자 동의
6. 아이 프로필 생성
7. 첫 15분 학습 시작

운영자는 진단의 가장 약한 연결과 첫 학습 진입 여부만 확인한다. 실제 답안을 마케팅 도구로 내보내지 않는다.

### D1~D2

- 재접속 여부
- 첫날에만 사용하고 멈춘 이유
- 음성·모바일·로그인 오류
- 아이가 혼자 시작할 수 있었는지

### D3

- 완료 학습일 3일 도달 여부
- 부모에게 확인 가능한 학습 기록 안내
- 가격 의향 질문 활성화
- 실제 최근 단어 하나를 부모가 아이에게 물어보도록 제안

### D4~D6

- 이틀 미학습 가정에 회복 메일
- 난이도·반복·학습시간 불편 수집
- 자동 난이도 조정을 약속하지 않음

### D7

- 실제 완료 학습일 수
- 진단에서 약했던 연결
- 최근 반복한 실제 단어 하나
- 12,900원 의향
- 15분 인터뷰 요청

D0↔D7 성적 향상, 정답률 변화, 자동 조정 결과는 현재 제공한다고 말하지 않는다.

이벤트를 추가할 때 아이 자유서술 답안과 개인식별자는 넣지 않는다.

## 6. WPA 계산

현재 정본:

```text
WPA = 가격 12,900원 사전 노출
  AND 진단을 아이 프로필에 연결
  AND completedLearningDays >= 3
  AND price_intent_answered.answer == yes
```

총 45분만으로 3일 조건을 대체하지 않는다. `streak`도 가격 질문과 7일 진행 판단에 사용하지 않는다.

## 7. 부모 연락

수동 이메일 원문은 `docs/OPEN_BETA_EMAILS.md`를 사용한다.

- D0: 진단 연결과 아이 프로필 생성 후
- D3: 완료 학습일 3일 도달 후
- Recovery: 이틀 미학습 후
- D7: 가입 7일차

치환값은 DB나 부모 대시보드에서 확인된 값만 쓴다. 확인하지 못한 누적시간, 점수 변화, 자동 난이도 조정 문장을 만들지 않는다.

## 8. 부모 인터뷰 15분

### 질문

1. 15Loop를 보기 전 아이 영어에서 가장 걱정한 장면은 무엇이었나요?
2. 무료 진단 결과 중 이해하기 쉬웠던 것과 어려웠던 것은 무엇인가요?
3. 아이가 첫 15분을 시작할 때 부모 도움이 얼마나 필요했나요?
4. 아이가 다시 사용한 날, 무엇이 재사용을 만들었나요?
5. 사용하지 않은 날에는 무엇이 막았나요?
6. 다른 단어장·앱·학원과 비교하면 무엇이 달랐나요?
7. 30일 12,900원을 검토하거나 거절한 가장 큰 이유는 무엇인가요?
8. 계속 쓰려면 꼭 고쳐야 할 한 가지는 무엇인가요?

### 기록 원칙

- 부모의 정확한 표현을 익명으로 기록한다.
- 아이 실명·학교·연락처를 콘텐츠에 사용하지 않는다.
- 칭찬만 모으지 말고 이탈과 거절 이유를 같은 비중으로 기록한다.
- 허락 없이 후기나 광고 인용으로 전환하지 않는다.

## 10. 주간 판정

### GO 신호

- 첫 학습 → 3일 완료 60% 이상
- WPA 5가정에 접근
- 아이가 부모 도움 없이 재사용하는 사례
- 부모가 진단 결과를 자기 아이 문제로 설명할 수 있음
- 실제 GPT-5.6 평가 성공률 95% 이상

### 수정 신호

- 진단 완료가 낮음 → 길이·설명·음성·모바일 검토
- 결과는 보지만 부모 연결이 낮음 → 결과 가치와 가입 허들 검토
- 첫날만 사용 → 난이도·재방문 장치·부모 안내 검토
- 사용은 하지만 가격 의향이 낮음 → 부모 증거와 가격 검토
- 가격 yes지만 아이가 3일 안 씀 → WPA에서 제외

## 11. ‘베타 1,000명’과의 관계

`베타 1,000명`은 Threads 공개 도전이나 Project JYP 콘텐츠 캠페인 목표로 사용할 수 있다. 하지만 운영 DB에서는 다음을 분리한다.

- Threads 도달
- 사이트 방문
- 진단 시작
- 진단 완료
- 부모 계정 생성
- 적합 가정
- 3일 완료 가정
- WPA

1,000명의 상단 숫자가 커져도 WPA가 늘지 않으면 15Loop 제품 검증은 성공이 아니다. 반대로 1,000명에 못 미쳐도 적합 30가정에서 WPA 5가정이 나오면 첫 제품 가설은 의미 있게 검증된다.

## 12. 첫 코호트 시작 전 체크리스트

- [ ] 가격 12,900원이 모집 링크 전에 보인다.
- [ ] UTM과 episode ID가 붙었다.
- [ ] 무가입 진단이 새 브라우저에서 정상 작동한다.
- [ ] Google·이메일 부모 로그인이 작동한다.
- [ ] 보호자 동의와 아이 프로필 생성이 작동한다.
- [ ] 재로그인 후 학습기록이 유지된다.
- [ ] GPT-5.6과 로컬 평가 출처가 구분된다.
- [ ] 운영자가 완료 학습일과 가격 의향을 확인할 수 있다.
- [ ] 수동 이메일 발신자와 회신 주소가 정해졌다.
- [ ] 인터뷰 동의와 익명 인용 원칙이 준비됐다.

## 13. 완료 정의

- 세 코호트가 분리돼 운영됐다.
- 모든 가정의 주요 이탈 단계가 기록됐다.
- WPA가 코드 기준으로 계산됐다.
- 5회 이상의 부모 인터뷰가 완료됐다.
- 실제 고객 언어가 마케팅 컨텍스트에 반영됐다.
- 다음 제품 결정이 조회수나 응원성 반응이 아니라 행동 데이터로 정해졌다.
